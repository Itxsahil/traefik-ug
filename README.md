# Traefik Reverse Proxy — UtkalGeeks

Traefik v3 acts as the reverse proxy and TLS terminator for all UtkalGeeks services.
It automatically issues and renews SSL certificates via Let's Encrypt, forces HTTPS on all traffic,
and strips the `www` prefix so all requests are served from `utkalgeeks.in`.

---

## File Structure

```
treafik/
├── docker-compose.yml   # Traefik service definition (single source of truth for config)
├── dynamic.yml          # Dynamic middleware config (www → non-www redirect)
├── treafik.yml          # Static config file (kept for reference, not mounted/used)
└── acme.json            # Auto-generated by Traefik — stores SSL certificates (do not edit)
```

> **Note:** `treafik.yml` is no longer mounted into the container. All static configuration
> is done via CLI flags in `docker-compose.yml` to avoid conflicts between the two sources.

---

## docker-compose.yml

### Image

```yaml
image: traefik:v3
```

Uses Traefik v3 (latest major version). Pinned to `v3` to avoid breaking changes from `latest`.

---

### Command flags (static configuration)

Each flag configures Traefik at startup.

#### Docker provider

```yaml
- "--providers.docker=true"
- "--providers.docker.exposedbydefault=false"
```

- Tells Traefik to watch Docker for containers to route to.
- `exposedbydefault=false` means a container is **ignored by Traefik unless** it has the label
  `traefik.enable=true`. This is a security best practice — containers opt in explicitly.

#### Entrypoints

```yaml
- "--entrypoints.web.address=:80"
- "--entrypoints.web.http.redirections.entrypoint.to=websecure"
- "--entrypoints.web.http.redirections.entrypoint.scheme=https"
- "--entrypoints.websecure.address=:443"
```

- `web` — listens on port 80 (HTTP). All traffic hitting port 80 is automatically redirected
  to `websecure` (HTTPS) with a 301 permanent redirect. No service ever receives plain HTTP.
- `websecure` — listens on port 443 (HTTPS). All actual traffic is served here.

#### Certificate resolver — Let's Encrypt

```yaml
- "--certificatesresolvers.letsencrypt.acme.httpchallenge=true"
- "--certificatesresolvers.letsencrypt.acme.httpchallenge.entrypoint=web"
- "--certificatesresolvers.letsencrypt.acme.email=admin@utkalgeeks.in"
- "--certificatesresolvers.letsencrypt.acme.storage=/acme.json"
```

- Defines a certificate resolver named `letsencrypt`.
- Uses the **HTTP-01 challenge** — Let's Encrypt sends a request to port 80 to verify domain
  ownership. This is why port 80 must remain open even though all user traffic is redirected.
- `email` is used by Let's Encrypt for expiry notifications.
- `storage` — certificates are persisted to `/acme.json` inside the container, which maps to
  `./acme.json` on the host (see volumes). **Do not delete this file** — it contains your
  issued certificates and rate limits apply if you request too many.

#### Traefik dashboard

```yaml
- "--api.dashboard=true"
```

Enables the Traefik web dashboard, accessible at `http://<server-ip>:8080`.
It shows all routers, services, middlewares, and certificate status in real time.

> **Security warning:** Port 8080 is exposed publicly. In production, either restrict it
> with a firewall rule or add authentication middleware to the dashboard router.

#### Dynamic config provider

```yaml
- "--providers.file.filename=/dynamic.yml"
```

Tells Traefik to load additional (dynamic) configuration from `/dynamic.yml` inside the
container, which maps to `./dynamic.yml` on the host. This is where middlewares that need
to be shared across services (like `redirect-to-root`) are defined.

### Ports

```yaml
ports:
  - "80:80"    # HTTP — used for Let's Encrypt HTTP challenge and global redirect to HTTPS
  - "443:443"  # HTTPS — all production traffic
  - "8080:8080" # Traefik dashboard
```

---

### Volumes

```yaml
volumes:
  - ./acme.json:/acme.json               # SSL certificate storage (persistent)
  - /var/run/docker.sock:/var/run/docker.sock:ro  # Docker socket (read-only)
  - ./dynamic.yml:/dynamic.yml:ro        # Dynamic middleware config (read-only)
```

- `acme.json` — mounted read-write so Traefik can store and update certificates.
  Must exist on the host before first run and have permissions `600`:
  ```bash
  touch acme.json && chmod 600 acme.json
  ```
- `docker.sock` — gives Traefik read access to Docker to discover containers and their labels.
  Mounted read-only for safety.
- `dynamic.yml` — mounted read-only. Traefik watches this file and hot-reloads on changes
  without needing a container restart.

---

### Network

```yaml
networks:
  traefik:
    external: true
```

Traefik connects to an external Docker network named `traefik`. All services that Traefik
should route to must also join this network. Create it once before first run:

```bash
docker network create traefik
```

## dynamic.yml

Defines shared middlewares that are loaded via the file provider.

### `redirect-to-root` middleware

```yaml
http:
  middlewares:
    redirect-to-root:
      redirectRegex:
        regex: "^https://www\\.utkalgeeks\\.in(.*)"
        replacement: "https://utkalgeeks.in${1}"
        permanent: true
```

- **What it does:** Strips the `www` prefix. Any request to `www.utkalgeeks.in` is permanently
  redirected (301) to `utkalgeeks.in`, preserving the full path and query string.
- **Regex:** Matches the full URL starting with `https://www.utkalgeeks.in`, captures
  everything after the domain (path, query string, etc.) into group `$1`.
- **Replacement:** Rebuilds the URL without `www`, appending `$1` to preserve the rest.
- **`permanent: true`:** Issues a 301 redirect. Browsers and search engines cache this,
  so `www` traffic is handled efficiently without hitting the app.

This middleware is applied to individual service routers via their labels, e.g.:
```yaml
- "traefik.http.routers.frontend.middlewares=redirect-to-root"
```

## acme.json

Auto-generated by Traefik on first run. Stores all Let's Encrypt certificates and account info.

- **Do not edit manually.**
- **Do not delete** in production — you will hit Let's Encrypt rate limits (5 certs per domain per week).
- **Must have `chmod 600`** — Traefik will refuse to use it otherwise.
- Should be backed up regularly.

## First-time Setup

```bash
# 1. Create the shared Docker network
docker network create traefik

# 2. Create acme.json with correct permissions
touch acme.json && chmod 600 acme.json

# 3. Start Traefik
docker compose up -d
```

## How a request flows

```
Browser: http://www.utkalgeeks.in/about
    │
    ▼
[Port 80 — web entrypoint]
    │  Global HTTP→HTTPS redirect (301)
    ▼
[Port 443 — websecure entrypoint]
    │  Matched by frontend router: Host(`www.utkalgeeks.in`)
    │  redirect-to-root middleware fires (301)
    ▼
https://utkalgeeks.in/about
    │  Matched by frontend router: Host(`utkalgeeks.in`)
    │  No middleware match — passes through
    ▼
[nginx container — serves React app index.html]
    │
    ▼
React Router handles /about client-side
```
